# 📝 腳本設計指南 — Script Design Guide

> 在進入 AI 圖片生成與影片製作之前，腳本設計是決定成品品質的關鍵環節。
> 本指南整合專業編劇流程與 AI 影片生成實務。

---

## 1. 腳本設計五步驟

```
Step 1: 需求萃取 ─→ Step 2: 節奏表 ─→ Step 3: 腳本撰寫 ─→ Step 4: 分鏡設計 ─→ Step 5: 腳本審查
```

### Step 1 ─ 需求萃取

從用戶輸入中提取關鍵要素：

```
□ 主題/核心概念：一句話描述
□ 目標平台：TikTok / Reels / Shorts / YouTube / 一般
□ 格式比例：9:16（預設）/ 16:9 / 1:1
□ 總時長：15s / 30s / 60s / 3min / 自訂
□ 視覺風格：寫實 / 動畫 / 賽博龐克 / 奇幻 / 復古 / 極簡
□ 情緒調性：歡樂 / 懸疑 / 感動 / 震撼 / 平靜
□ 參考素材：圖片 / 影片 / 音樂（可選）
□ 特殊要求：角色一致性 / 產品展示 / 特定色彩
```

### Step 2 ─ 節奏表 (Beat Sheet)

根據總時長選擇對應節奏結構：

**15 秒（極短）：**
```
Hook (0-3s) → 核心展示 (3-10s) → 反轉/驚喜 (10-13s) → 收尾 (13-15s)
```

**30 秒（標準短影片）：**
```
Hook (0-3s) → Setup (3-8s) → 衝突 (8-15s) → 高潮 (15-25s) → Outro (25-30s)
```

**60 秒（深入短影片）：**
```
Hook (0-5s) → 情境建立 (5-15s) → 情感連結 (15-25s) → 衝突 (25-35s) → 解決 (35-45s) → 高潮 (45-55s) → CTA (55-60s)
```

**3 分鐘（故事型）：**
```
第一幕 (0-45s):  開端 → 角色建立 → 催化事件
第二幕 (45-135s): 行動 → 中點 → 危機
第三幕 (135-180s): 高潮 → 結局
```

#### 情緒曲線

設計整支影片的情緒起伏：

```
        ▲ 高潮
        │    ╱╲
        │   ╱  ╲
        │  ╱    ╲
        │ ╱      ╲
  引入 ─┼╱        ╲
        │          ╲
        │           ╲
  平靜 ─┼────────────╲───────
        └────────────────────
        0s    中點    結尾
```

### Step 3 ─ 腳本撰寫

將節奏表轉化為文字腳本：

```
【場景 1 — 標題】
時長: 8s | 情緒: 平靜 | 轉場: dissolve

畫面描述：
[鏡頭角度 + 主體 + 環境 + 光線 + 色調]

對白/旁白：
[如有]

動態描述（給 AI 影片用）：
[主體動作 + 環境動態 + 鏡頭運動]
```

#### 腳本撰寫原則

| 原則 | 說明 |
|------|------|
| **視覺化寫作** | 用畫面思考，非文字思考 |
| **精簡** | 每場景一句話描述核心 |
| **可視性** | 確認每個描述都能被 AI 模型理解 |
| **連續性** | 前後場景的視覺元素一致 |
| **節奏感** | 長短場景交錯，避免單調 |

### Step 4 ─ 分鏡設計

將腳本拆解為逐鏡頭分鏡，每個分鏡包含：

```
Scene #    | 0           | 1           | 2
───────────|─────────────|─────────────|─────────────
時長       | 0-8s        | 8-18s       | 18-30s
視覺       | [keyframe]  | [keyframe]  | [keyframe]
運鏡       | push-in     | orbit       | pull-out
情緒       | 平靜        | 緊張        | 爆發
轉場       | -           | dissolve    | cut
```

#### Image Prompt 建構公式（給 Agnes Image 2.1 Flash）

```
[主體] in [場景], [姿勢/表情],
cinematic composition, [光線], [風格],
clean sharp details, no motion blur, well-lit
```

#### Video Prompt 建構公式（給 Agnes Video 2.0）

```
The camera [motion] as the subject [action].
[environmental motion].
[timing/sequence].
```

#### Video Negative Prompt 預設

```
blurry, low quality, jpeg artifacts, flickering,
inconsistent character, morphing, disfigured face,
excessive motion, shaky cam, hyper,
cartoon, 3d render, anime, cgi, oversaturated
```

➡ **關鍵原則：** 圖片鎖定視覺品質，提示詞只描述動態。

### Step 5 ─ 腳本審查

最終檢查清單：

```
□ 連續性檢查
   □ 主角外觀一致（服裝/髮型/體型）
   □ 場景方向感一致
   □ 光線/色調延續合理
   □ 情緒曲線有邏輯
   □ 轉場方式符合內容調性
   □ character_card 在每個場景中一致

□ 負面提示詞檢查
   □ 每個場景都有 image_negative_prompt
   □ 每個場景都有 video_negative_prompt
   □ 負面提示詞涵蓋品質/一致性/風格偏移

□ 模型匹配檢查
   □ Image prompt 不含「動態」描述
   □ Video prompt 不重複圖片已有的內容
   □ 場景秒數在 5-15s 範圍
   □ 總時長符合目標平台規範

□ 可行性檢查
   □ 每個場景的描述對 AI 模型是可理解的
   □ 非人體/非限制級（除非走 NSFW 路由）
   □ 至少 2 個場景（單場景無法串連）
```

## 2. 腳本模板參考

### 產品展示型

```
Hook (3s):      產品特寫 + 問題痛點
情境建立 (7s):  使用場景展示
特色展示 (10s): 3 個核心功能
價值證明 (7s):  前後對比 / 使用效果
CTA (3s):       引導行動
```

### 故事型（三幕劇）

```
第一幕 (25%): 主角日常 → 催化事件
第二幕 (50%): 探索/挑戰 → 中點轉折 → 谷底
第三幕 (25%): 覺醒 → 高潮 → 新常態
```

### 教學/知識型

```
Hook (3s):     驚人事實 / 問題
核心內容 (15s): 3 個要點
示範 (10s):    實際操作
總結 (5s):     重點回顧
CTA (2s):      追蹤/收藏
```

## 3. AI 影片腳本特殊考量

### 圖片需為「可動的」

| 要素 | 建議 |
|------|------|
| 主體姿勢 | 自然可動（非誇張到無法延續） |
| 背景 | 有深度層次（前中後景） |
| 動態暗示 | 飄動衣物、未定動作、粒子 |
| 構圖 | 留邊讓運鏡有空間 |

### 轉場類型

| 轉場 | 適用情境 | 注意 |
|------|----------|------|
| cut | 時間跳躍 / 風格切換 | 最簡單，但生硬 |
| dissolve | 時間流逝 / 夢境 | 平滑，適合 AI 模型 |
| wipe | 場景變換 | 需明確方向 |
| morph | 奇幻變形 | 最難，但效果驚豔 |
