# 🏗️ AI Media Pipeline — 流水線架構

## 三階段流程

### 全景圖

```
┌────────────────────────────────────────────────────────────────────────┐
│                        多模態流水線 (Multimodal Pipeline)                │
├────────────────────────────────────────────────────────────────────────┤
│                                                                        │
│  Phase 0                Phase 1                Phase 2                 │
│  腳本設計                圖片生成                影片製作                  │
│                    (Multimodal I2V)                                    │
│                                                                        │
│  文字 → 結構化腳本    文字 + 風格 → 圖片    圖片 + 動態 → 影片          │
│                                                                        │
│  ┌──────────┐       ┌──────────────┐      ┌──────────────┐           │
│  │ Beat     │       │ Agnes 2.1    │      │ Agnes Video  │            │
│  │ Sheet    │ ──→   │ Flash (圖)   │ ──→  │ 2.0 (I2V)    │           │
│  │          │       │              │      │              │            │
│  │ 腳本     │       │ SD Forge     │      │ Frame        │            │
│  │ 撰寫     │       │ (NSFW)       │      │ Chaining     │            │
│  └──────────┘       └──────────────┘      └──────────────┘           │
│                                                                        │
│  ↑ 多模態：文字→結構   ↑ 多模態：文字→圖片    ↑ 多模態：圖片+文字→影片    │
│                                                                        │
└────────────────────────────────────────────────────────────────────────┘
```

### 模型參數與限制

| 模型 | 用途 | Temperature | Max Tokens | 其他限制 |
|------|------|-------------|------------|----------|
| **Agnes 2.0 Flash** | 腳本/推理 | 0.7 (預設) | 4,096 | — |
| **Agnes Image 2.1 Flash** | 圖片生成 | 不適用 (擴散模型) | — | 解析度限制: 1024x1792 max, 4,000張/日 |
| **Agnes Video 2.0** | 影片生成 | 不適用 (擴散模型) | — | 幀數 8n+1, 500秒/日, 1080p max 169幀 |
| **SD Forge (DS8/RV6)** | NSFW 生圖 | CFG Scale: 7(DS8)/5(RV6) | — | Steps: 25, 本機 VRAM 4GB |

> **Temperature 規則：** 腳本生成用 0.7（平衡創意與結構化）；若輸出格式不穩（JSON parse 失敗），降溫至 0.3。
> **擴散模型：** 圖片/影片模型是 diffusion-based，無 temperature 參數，改用 CFG Scale 控制創造力。

### Phase 0：前置（用戶輸入）

```
輸入: 主題 / 風格 / 長度 / 平台 / 參考素材
  │
  ├─ 主題描述（必要）
  ├─ 視覺風格建議（可選，預設自動）
  ├─ 目標平台（抖音/Reels/Shorts/一般）
  ├─ 總時長（可選，預設由腳本決定）
  └─ 參考圖片/影片（可選）
```

### Phase 1：腳本設計（新增）

```
  ┌─────────────────────────────────────────┐
  │  Step 1: 需求萃取                          │
  │  · 解析用戶輸入 → 主題關鍵字 + 風格向量      │
  │  · 確認目標平台格式（9:16/16:9/1:1）        │
  └──────────────┬──────────────────────────┘
                 ↓
  ┌─────────────────────────────────────────┐
  │  Step 2: 節奏表 (Beat Sheet)              │
  │  · 三幕劇結構 / 短影片鉤子結構              │
  │  · 時間軸分配（起承轉合）                   │
  │  · 情緒曲線設計（高→低→高）                │
  └──────────────┬──────────────────────────┘
                 ↓
  ┌─────────────────────────────────────────┐
  │  Step 3: 腳本撰寫                          │
  │  · 場景描述 + 視覺風格                     │
  │  · 對白/旁白（若有）                       │
  │  · 轉場設計                               │
  └──────────────┬──────────────────────────┘
                 ↓
  ┌─────────────────────────────────────────┐
  │  Step 4: 分鏡設計                          │
  │  · 逐鏡頭拆解（scene_id）                  │
  │  · 每個鏡頭: image_prompt + video_prompt  │
  │  · 時長分配 (5-15s per scene)             │
  └──────────────┬──────────────────────────┘
                 ↓
  ┌─────────────────────────────────────────┐
  │  Step 5: 腳本審查                          │
  │  · 連續性檢查（角色/環境/方向）              │
  │  · 時長總和驗證                            │
  │  · 可行性評估（與模型能力匹配）               │
  └──────────────────────────────────────────┘
                 ↓
           輸出: script_package.json
```

### Phase 2：圖片生成

```
  script_package.json
         ↓
  ┌─────────────────────────────────────────┐
  │  路由判斷                                   │
  │  · NSFW 內容？→ SD Forge (本機)            │
  │  · 一般圖片？→ Agnes Image 2.1 Flash      │
  └────────────────┬────────────────────────┘
                   ↓
  ┌─────────────────────────────────────────┐
  │  批次生圖 (async parallel)               │
  │  · 每場景一張 keyframe                   │
  │  · 解析度 1024x1792 (9:16)               │
  │  · 4 次重試 + 指數退避                    │
  └────────────────┬────────────────────────┘
                   ↓
           輸出: image_jobs.json + scenes/
```

### Phase 3：影片製作

```
  script_package.json + scenes/
         ↓
  ┌─────────────────────────────────────────┐
  │  模式選擇                                   │
  │  · --multi-image → 多圖轉場 (單次 API)     │
  │  · 一般模式 → 逐圖 I2V                     │
  └────────────────┬────────────────────────┘
                   ↓
  ┌─────────────────────────────────────────┐
  │  影片生成                                   │
  │  · Agnes Video 2.0                       │
  │  · num_frames + frame_rate 控制長度       │
  │  · 異步輪詢 (5s interval)                 │
  └────────────────┬────────────────────────┘
                   ↓
  ┌─────────────────────────────────────────┐
  │  Frame Chaining (可選)                    │
  │  · 前一鏡頭末幀 → 下一鏡頭起始圖片           │
  └──────────────────────────────────────────┘
                   ↓
           輸出: video_jobs.json + videos/
```

## 狀態機

```
INIT → SCRIPT_DONE → IMAGE_GEN → IMAGE_DONE → VIDEO_GEN → VIDEO_DONE → COMPLETE
                                                                    ↓
                                                               (可中斷續傳)
```

- 狀態存於 `agents_workflow_state.json`
- 斷電/中斷後重跑自動恢復
- `--reset` 重置所有狀態

## 一致性控制

### 人物一致性鎖定

跨場景保持角色特徵一致：

```
角色描述卡 (鎖定):
- 性別 / 年齡 / 體型
- 服裝款式與顏色
- 髮型與髮色
- 種族/膚色
- 主要特徵（眼鏡、配件、刺青等）
```

**實務做法：**
1. 在 `script_package.json` 的每個 scene 中嵌入角色描述卡
2. Image prompt 中**重複**角色關鍵特徵（不要假設模型會記住）
3. Video prompt 中描述主體時提到角色（e.g., "the woman in the plastic raincoat"）

### 背景一致性鎖定

| 面向 | 鎖定方式 |
|------|----------|
| 場景 | 每個場景的 visual_style 欄位描述完整環境 |
| 色調 | 跨場景的色調曲線保持一致 |
| 光源方向 | 同一場景內光源方向統一 |
| 時代設定 | 明確在 scene_goal 中標註 |

### 連續性檢查

```
□ 主角外觀一致（服裝/髮型/體型）
□ 場景方向感一致
□ 光線/色調延續合理
□ 情緒曲線有邏輯
□ 轉場方式符合內容調性
□ 角色在不同場景中特徵一致
□ 背景元素不憑空出現/消失
```

## 多模態工作流

### 原則

```
Phase 0 (文字)       →   文字輸入 → 結構化腳本輸出
Phase 1 (文字→圖片)   →   文字 prompt → 視覺 keyframe
Phase 2 (圖片+文字→影片) → keyframe + 動態描述 → 影片
```

**為什麼圖片優先？**
1. 圖片鎖定構圖、主體、光影、風格 — 這些是影片的視覺基礎
2. 影片模型（Agnes Video 2.0）是 I2V（Image-to-Video）架構，圖片品質直接決定影片品質
3. Frame Chaining 需要上一個鏡頭的圖片作為下一個鏡頭的起點

### 不支援的做法

```
❌ 純文字直接生影片（T2V）— 品質不穩定，無法控制構圖
❌ 先有影片再抽圖 — 順序反了
❌ 多個場景共用同一張圖 — 除非刻意設計
```

## API 配額

| 服務 | 配額 | 週期 |
|------|------|------|
| Agnes 文字 | 1,500 req | 5h |
| Agnes 圖片 | 4,000 張 | 日 |
| Agnes 影片 | 500 秒 | 日 |
| SD Forge | 無限制 | 本機 |

## 模型版本鎖定

| 組件 | 版本 | 原因 |
|------|------|------|
| 圖片 (non-NSFW) | Agnes Image 2.1 Flash | 品質與速度平衡 |
| 影片 | 2.0 架構 (agnes-video-v2.0) | 政策鎖定，非舊版 |
| NSFW | SD Forge (DreamShaper 8 / RV V6) | 雲端不支援成人內容 |
| 腳本推理 | Agnes 2.0 Flash | 成本效率最佳 |
