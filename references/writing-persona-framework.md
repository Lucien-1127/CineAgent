# CineAgent Writing Persona Framework
## 寫作人格 × 敘事技法 × 技術文件

> 一份融合產品寫作聲音（Product Writing Voice）、劇本敘事技法（Scriptwriting Techniques）與台灣在地技術寫作習慣的實用框架。
> 適用場景：README、文件、社群貼文、錯誤訊息、產品公告。

---

## 一、什麼是「寫作人格」

寫作人格是產品的「文字面龐」。三個層次：

| 層次 | 說明 | 例子 |
|------|------|------|
| **Voice（聲音）** | 穩定不變的核心人格 | 直接、務實、有溫度 |
| **Tone（語調）** | 依情境調整的表達 | 教學時耐心，錯誤時坦率 |
| **Style（風格）** | 具體的文字選擇 | 用詞偏好、emoji 使用 |

> Google Developer Documentation Style Guide: "Be conversational and friendly without being frivolous."

---

## 二、頂級產品的寫作聲音

| 產品 | 類型 | 關鍵技巧 |
|------|------|----------|
| **Stripe** | 清晰權威 | 用「you」直接對話，先結論再細節 |
| **Linear** | 極簡自信 | 用詞極少但精準，不解釋自己為什麼好 |
| **Vercel** | 先鋒夥伴 | 用「Imagine...」開啟敘事，把部署包裝成解放 |
| **Tailwind** | 務實導師 | 直接給範例再解釋，誠實列出限制 |
| **Notion** | 溫暖夥伴 | 用「你」直接呼喚，強調可能性 |
| **Resend** | 極簡主義 | 一句話說一件事，毫不拖泥帶水 |

## 三、劇本敘事 × 技術文件融合

### 五幕結構映射

| 劇本元素 | 文件對應 | 實作 |
|----------|----------|------|
| Hook | README 開頭「Why」 | 用痛點問題開場 |
| Inciting Incident | 安裝/Quick Start | 5 分鐘內看到成果 |
| Rising Action | 核心功能教學 | 由淺入深 |
| Climax | 進階應用 | 展示最強能力 |
| Resolution | 下一步指引 | 明確告訴讀者接下來做什麼 |

### 場景節奏

```
慢（建立情境）→ 快（動作指令）→ 停（確認理解）→ 快（下一個動作）→ 慢（反思總結）
```

### 敘事角色

| 角色 | 功能 | 語氣 |
|------|------|------|
| **Guide（導師）** | 文件本身 | knowledgeable but humble |
| **Hero（英雄）** | 使用者 | 有能力解決問題 |
| **Villain（反派）** | 複雜的影像製作流程 | 具體、可感知 |

### 情感弧線

```
困惑 → 好奇 → 自信 → 渴望
```

## 四、台灣在地化

### 寫作習慣

| 面向 | 台灣偏好 |
|------|----------|
| 代名詞 | 你 / 我們 |
| 術語 | 中英混用（deploy pipeline 比「部署管線」自然） |
| 語氣 | 直接但不粗魯 |
| 優先級 | 先跑起來再看原理 |
| 範例 | 最小可驗證片段 |

### CineAgent 規則

- ✅ 用「你」而不是「我們」
- ✅ 短句、主動語態、現在時
- ✅ 中英混用是常態，繁體中文為主
- ✅ 用 emoji 适度增强可读性（🎬 🤖 ✨）
- ❌ 不用八股結構（首先、其次、最後）
- ❌ 不說空話套話（我們致力於、我們相信）
- ❌ 不對使用者說教（你應該要知道）
- ❌ 不預先宣告內容（接下來我們會介紹）

## 五、CineAgent 寫作人格卡

```
🎯 核心定位: 一位精通 AI 影像製作的台灣工程師朋友
🏷️ 五大性格: 直接、務實、自信、有溫度、有品味
💬 核心 Voice: 直接、務實、有溫度
🎬 敘事角色: 導師 = 文件 / 英雄 = 你 / 反派 = 複雜流程

Good:  「python run_pipeline.py --topic '你的主題' 就搞定了。」
Bad:   「我們建議您使用我們的產品來實現影像生成的功能。」

Good:  「出錯了？檢查一下 API key 有沒有設對。」
Bad:   「若發生錯誤，請確認您的憑證是否正確無誤。」
```
